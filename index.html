Instalação

Instale em 3 passos

1
Baixe o instalador
Descarregue o ficheiro NexDock-Setup.run e dê permissão de execução.
# Dê permissão de execução chmod +x NexDock-Setup.run # Execute o instalador gráfico ./NexDock-Setup.run
2
Siga o wizard
O instalador abre automaticamente com a interface gráfica. Clique em Próximo → Próximo → Instalar — sem configuração necessária.
3
Abra o NexDock
Clique no ícone do Desktop ou execute o comando abaixo.
# Inicia o engine e abre a UI nexdock daemon start nexdock ui # Ou use o atalho criado no Desktop # Duplo clique em: NexDock.desktop

nexdock-compose

Suba stacks completas
com um ficheiro

Compatível com docker-compose.yml. Defina MySQL + phpMyAdmin + Redis numa só linha e inicie tudo com um comando.

# Sobe toda a stack nexdock-compose up # Para tudo nexdock-compose down
nexdock-compose.yml
services:   mysql: image: mysql:8.0 environment: - MYSQL_ROOT_PASSWORD=secret - MYSQL_DATABASE=myapp ports: - "3306:3306" restart: always   phpmyadmin: image: phpmyadmin:latest environment: - PMA_HOST=localhost - PMA_PASSWORD=secret ports: - "8080:80"   redis: image: redis:7 ports: - "6379:6379"

FAQ

Perguntas frequentes

É mesmo grátis? Sem limites? +
Sim, 100% gratuito e open source. Sem limite de containers, sem limite de imagens, sem subscrição, sem conta obrigatória. Pode usar para projectos pessoais e comerciais.
Preciso de desinstalar o Docker primeiro? +
Não. O NexDock funciona em paralelo com o Docker se quiser. Usa portas diferentes (7474 para a API, 7475 para a UI) e um directório separado (~/.nexdock). Os dois podem coexistir sem conflitos.
Funciona no macOS ou Windows? +
Actualmente o NexDock é exclusivo para Linux, pois usa Linux Namespaces e cgroups — a mesma tecnologia base do Docker. Suporte para macOS (via Lima/QEMU) está planeado para versões futuras.
Posso usar imagens que já tenho no Docker? +
O NexDock usa o Docker Hub directamente e baixa as imagens no seu próprio formato. Não reutiliza as imagens do Docker local, mas pode baixar as mesmas imagens. O processo é automático — escolhe a imagem no catálogo e clica em baixar.
Como funciona o isolamento de containers? +
O NexDock usa chroot para isolamento do sistema de ficheiros e unshare (Linux Namespaces) para isolamento de processos quando executado com permissões adequadas. Em modo utilizador normal, usa chroot sem root — suficiente para desenvolvimento local.
Posso contribuir ou reportar bugs? +
Claro! O NexDock é open source. Pode reportar problemas, sugerir funcionalidades ou contribuir com código. Todas as contribuições são bem-vindas.
